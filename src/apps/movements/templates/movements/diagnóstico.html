Solución de Problemas de Inventario

Diagnóstico y solución para la duplicación de entradas y errores en el registro de movimientos

Problema Identificado
Duplicación de entradas y registro incorrecto

Al registrar una entrada de productos, el sistema está:

  • Duplicando la cantidad en el inventario
  • Mostrando entradas como salidas en algunos casos
  • Actualizando incorrectamente los totales
Posibles causas en tu código:
  • Falta de prevención de envíos duplicados del formulario
  • Problemas con la función calculateTotal() en movement_form.html
  • Validación insuficiente en el backend para movimientos
  • Posible error en la API de información de productos

1 Prevenir envíos duplicados del formulario

Agrega este código a tu archivo movement_form.html para evitar que el formulario se envíe múltiples veces:

Código para movement_form.html
// Prevenir doble envío del formulario
document.querySelector('#movement-form').addEventListener('submit', function(e) {
    const submitBtn = this.querySelector('button[type="submit"]');
    submitBtn.disabled = true;
    submitBtn.innerHTML = '<i class="fas fa-spinner fa-spin"></i> Procesando...';
});
2 Corregir la función calculateTotal()

Reemplaza la función calculateTotal() en movement_form.html con esta versión mejorada:

Código para movement_form.html
// Función para calcular el valor total (versión corregida)
// Función para calcular el valor total (versión corregida)
function calculateTotal() {
    const quantity = parseFloat(quantityInput.value) || 0;
    const unitPrice = parseFloat(unitPriceInput.value) || 0;
    
    // Validar valores negativos
    if (quantity < 0 || unitPrice < 0) {
        totalValueElement.textContent = 'Error: Valores negativos';
        totalValueElement.style.color = '#dc3545';
        return;
    }
    
    const total = quantity * unitPrice;
    totalValueElement.textContent = '$' + total.toLocaleString('es-VE', {
        minimumFractionDigits: 2,
        maximumFractionDigits: 2
    });
    totalValueElement.style.color = '#0d6efd';
}

Implementar Soluciones

Para resolver los problemas de duplicación en tu sistema, sigue estos pasos:

  1. Abre tu archivo movement_form.html
  2. Localiza la etiqueta <script> al final del archivo
  3. Reemplaza la función calculateTotal() con la versión corregida proporcionada arriba
  4. Añade el código de prevención de envíos duplicados
  5. Agrega la validación adicional de campos requeridos
  6. Guarda los cambios y prueba el formulario
Tip: Después de implementar estas correcciones, verifica también tu código backend para asegurarte de que no haya lógica duplicada que actualice el inventario.
